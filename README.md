## Problem Statement

Organizations face challenges managing tenders efficiently and transparently. Traditional systems suffer from lack of transparency, susceptibility to corruption, and manual error-prone contract execution.

TenderChain solves this using immutable blockchain records, smart-contract-enforced rules, and role-based access control.

## Architecture Overview

```
User (MetaMask) --> React Frontend --> ethers.js --> Hardhat Local Node
                                                          |
                                                 TenderManagement.sol
```

Blockchain Type: Private Ethereum (Hardhat Network, Proof of Authority)
Consensus: Proof of Authority (Clique) — chosen for enterprise use where validators are known and trusted, with high throughput and deterministic finality.

## Smart Contract Functions

### User Management
- selfRegister(name, org, role) — Self-register as Tenderer or Bidder
- registerUserByAdmin(addr, name, org, role) — Admin registers a user
- getUser(addr) — Get user profile

### Tender Management
- createTender(title, desc, req, budget, deadline, criteria[]) — Create and publish a tender
- closeTender(tenderId) — Close tender to new bids
- cancelTender(tenderId) — Cancel tender, reject all bids
- getTender(tenderId) — Read tender details
- getAllTenderIds() — Get all tender IDs
- getUserTenders(addr) — Get tenders by tenderer

### Bid Management
- submitBid(tenderId, amount, proposal, technical, days) — Submit a bid
- evaluateBid(bidId, score) — Score a bid from 0 to 100
- awardTender(tenderId, bidId) — Award tender and auto-create contract
- getBid(bidId) — Read bid details
- getTenderBids(tenderId) — Get all bid IDs for a tender
- getUserBids(addr) — Get bids submitted by a user

### Contract Management
- completeContract(contractId) — Mark contract as completed
- raiseDispute(contractId) — Raise a dispute on a contract
- resolveDispute(contractId, markComplete) — Admin resolves a dispute
- getContract(contractId) — Read contract details
- getAllContractIds() — Get all contract IDs
- getTenderContractId(tenderId) — Get contract linked to a tender

## Setup and Running

### Step 1 — Install root dependencies
```bash
cd blockchain-tender-system
npm install
```

### Step 2 — Compile the smart contract
```bash
npx hardhat compile
```

### Step 3 — Start Hardhat local node (Terminal 1)
```bash
npx hardhat node
```
Copy Account #0's private key — this is the Admin wallet.

### Step 4 — Deploy the contract (Terminal 2)
```bash
npx hardhat run scripts/deploy.js --network localhost
```
This writes the ABI and contract address to frontend/src/contracts/ automatically.

### Step 5 — Start the React frontend
```bash
cd frontend
npm install
npm start
```
Opens at http://localhost:3000

### Step 6 — Configure MetaMask
Add a custom network:
- Network Name: Hardhat Local
- RPC URL: http://127.0.0.1:8545
- Chain ID: 31337
- Currency Symbol: ETH

Import Account #0 as Admin. Import 2-3 more accounts as Tenderer and Bidder.

## Usage Guide

1. Connect MetaMask on the landing page
2. Account #0 (Admin) is auto-registered
3. Register other accounts as Tenderer or Bidder via the Register page
4. Tenderer creates a tender at /create-tender
5. Bidders submit bids on the tender detail page
6. Tenderer closes the tender, evaluates bids with scores, and awards to the best bid
7. A contract is auto-created on award
8. Tenderer marks contract complete; either party can raise a dispute
9. Admin resolves disputes from /admin

## Network and Consensus

Network: Private Ethereum (Hardhat local node)
Consensus: Proof of Authority (Clique PoA)

Justification: Enterprise tender systems are permissioned with known validators (government bodies, approved organizations). PoA provides instant deterministic finality required for legal contract creation, high throughput with ~1s block time, and no energy-wasting mining. This aligns with production deployments on Hyperledger Besu or Quorum.

## Security Analysis

| Threat                  | Mitigation                                      |
|-------------------------|-------------------------------------------------|
| Unauthorized actions    | Role-based modifiers on every function          |
| Duplicate bids          | On-chain duplicate check in submitBid           |
| Bid after deadline      | block.timestamp check                           |
| Score manipulation      | Only tender creator or Admin can evaluateBid    |
| Re-entrancy             | No ETH transfers; state updated before events   |
| Integer overflow        | Solidity 0.8.x built-in overflow checks         |
| Unauthorized completion | Party/ownership verified on contract actions    |
